## 7.1 场景描述与痛点

### 真实痛点

> "每次月结对账都是噩梦——银行对账单和账套记录来回核对，眼睛都看花了，还经常漏掉问题。" —— 财务从业者

### 痛点一览

| 痛点 | 具体表现 | 影响 |
|------|---------|------|
| **人工核对耗时** | 逐笔对比银行流水和账套记录 | 每次对账3-5天 |
| **差异难发现** | 金额、时间稍微不同就漏掉 | 导致账实不符 |
| **重复劳动多** | 每月重复同样的对账流程 | 没有技术含量但必须做 |
| **归档麻烦** | 对账结果需要整理归档 | 额外工作量 |

## 7.2 用 WorkBuddy 自动对账

### 方案一：银行对账单 vs 账套记录

把两份数据都扔给 WorkBuddy：

```
对比以下两组数据，找出差异项：

数据A（银行对账单）：
[粘贴或上传银行流水]

数据B（财务系统记录）：
[粘贴或上传账套数据]

匹配规则：
- 日期相同（允许±3天误差）
- 金额相同（完全相等）

未匹配的交易按以下方式分类：
1. 银行已收企业未收
2. 银行已付企业未付
3. 企业已收银行未收
4. 企业已付银行未付

输出：
- 匹配结果汇总（总数、匹配数、差异数）
- 未匹配项明细表
- 差异金额汇总
```

---

### 方案二：应收账款核对

核对客户欠款：

```
核对以下客户应收款：

客户名称：[客户名]
应收明细：[粘贴数据]

核销记录：[粘贴收款记录]

1. 计算每笔应收款的账龄
2. 找出超期未收的款项
3. 计算已收未核销、应收未核销的情况
4. 生成客户对账单

输出：
- 账龄分析表（30天内/60天内/90天内/超期）
- 超期催款建议
```

---

### 方案三：应付账款核对

核对供应商付款：

```
核对以下供应商应付款：

供应商：[供应商名]
应付明细：[粘贴数据]

付款记录：[粘贴付款流水]

1. 核对每笔付款是否正确核销
2. 找出多付、少付的情况
3. 找出已付供应商未确认的款项
4. 生成供应商对账函

输出：
- 应付核对结果
- 差异项及建议处理方式
- 供应商对账单草稿
```

---

### 方案四：三单匹配（发票-入库单-付款）

核对采购三单是否一致：

```
核对以下采购三单：

入库单：[粘贴数据]
发票：[粘贴数据]
付款记录：[粘贴数据]

匹配规则：
1. 入库单金额 = 发票金额（含税）
2. 付款金额 ≤ 发票金额
3. 三单日期顺序：入库 → 开票 → 付款

找出：
1. 金额不一致的三单
2. 发票已收但未入库
3. 付款金额大于发票金额
4. 已付款但未收到发票

输出：差异清单 + 处理建议
```

## 7.3 实战案例

### 案例：月末银行对账

**任务背景**：每月末需要核对工商银行账户的银行对账单和账套记录，涉及约200笔交易。

**传统方式**：
1. 导出银行对账单（30分钟）
2. 导出账套记录（30分钟）
3. 逐笔对比（8小时）
4. 找出差异项（2小时）
5. 编制银行调节表（1小时）

**用 WorkBuddy**：

```
第一步：核对交易
"对比附件中的银行对账单和账套记录，
找出所有未匹配的交易，
按'银行已收企业未收/企业已收银行未收'分类，
计算差异金额"
（10分钟得到核对结果）

第二步：生成调节表
"基于以上核对结果，生成银行余额调节表：
- 银行对账单余额
- 加：企业已收银行未收
- 减：企业已付银行未付
- 调整后余额
- 账套余额
- 差异"
（5分钟得到调节表草稿）

第三步：差异分析
"分析这些未匹配项：
- 日期在5号之前的交易有X笔
- 金额超过5万元的有X笔
- 列出最需要关注的3笔差异"
（5分钟得到分析报告）
```

**结果**：30分钟完成原本3天的工作。

## 7.5 常见问题

### Q1：数据格式不统一无法匹配？

先让 WorkBuddy 统一格式：

```
清洗这两份数据，统一格式后再核对：
- 日期格式统一为 YYYY-MM-DD
- 金额去掉千分位
- 摘要文字去掉空格
```

### Q2：差异项太多怎么办？

让 WorkBuddy 帮你分级：

```
分析这些差异项：
1. 按金额排序，列出Top10大额差异
2. 按日期分类，找出规律
3. 判断哪些是正常时差、哪些需要跟进
```

### Q3：如何生成银行调节表？

直接说：

```
基于以上核对结果，生成银行余额调节表：
- 银行对账单余额：XXX
- 加：企业已收银行未收（X笔）
- 减：企业已付银行未付（X笔）
- 调整后余额：XXX
- 账套余额：XXX
- 差异：XXX
```
